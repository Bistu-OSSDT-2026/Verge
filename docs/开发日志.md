## 一、路径系统（PathVisual）

### 当前路径布局

路径由 17 个 `ColorRect`（60×60，2px 内缩，形成 4px 深色缝隙作为网格线）组成，位于 `PathVisual` 节点下。

**网格规格**：标准 64×64 网格，每个格子视觉尺寸 60×60（四周留 2px 间隙）。

**路径走向**（敌人移动路线，来自 `enemy_movement.gd`）：
```
起点 (32, 320)
  → 向右 → (96,320) → (160,320) → (224,320) → (288,320) → (352,320)
  → 向下拐弯 → (384,384)
  → 向右 → (448,448) → (512,448) → (576,448) → (640,448) → (704,448)
  → 向上拐弯 → (768,384)
  → 向上 → (768,352) ← 核心位置（路径终点）
```

**PathVisual 节点列表**（均在 `main_game.tscn`）：

| 节点名 | 格子坐标 | offset_left | offset_top | 说明 |
|--------|---------|-------------|-----------|------|
| PathH1_G0~G5 | (32~352, 320) | 2+32n | 290 | H1 段 6 格 |
| PathCorner1 | (384, 320) | 354 | 290 | 右下拐弯角 |
| PathV1_G1 | (384, 384) | 354 | 354 | 垂直连接 |
| PathCorner2 | (384, 448) | 354 | 418 | 右拐弯角 |
| PathH2_G0~G4 | (448~704, 448) | 418+64n | 418 | H2 段 5 格 |
| PathCorner3 | (768, 448) | 738 | 418 | 上拐弯角 |
| PathV2_G1 | (768, 384) | 738 | 354 | 垂直连接 |
| PathCoreCell | (768, 320) | 738 | 290 | 核心格子（灰色背景） |

- **背景色**：`Color(0.05, 0.05, 0.08, 1)`（深色，衬托灰色路径格）
- **路径格颜色**：`Color(0.38, 0.38, 0.44, 1)`（灰色）

## 三、部署格子（DeploySlots）

### 总数：19 个（Slot0 ~ Slot18）

**16 个地面格（ground）** — 放在路径格子上，可以部署地面角色和地雷：

| Slot | 位置 (x, y) | 对应路径格 |
|------|-------------|-----------|
| Slot0~5 | (32,96,160,224,288,352, 320) | H1 段 |
| Slot6 | (384, 320) | Corner1 |
| Slot7 | (384, 384) | V1_G1 |
| Slot8 | (384, 448) | Corner2 |
| Slot9~13 | (448,512,576,640,704, 448) | H2 段 |
| Slot14 | (768, 448) | Corner3 |
| Slot15 | (768, 384) | V2_G1 |

**3 个高台格（high）** — 不在路径上，只能部署高台角色（狙击手）：
- Slot16: `(160, 224)`
- Slot17: `(288, 224)`
- Slot18: `(576, 352)`

### 高台格特殊逻辑

- 常态下**无任何视觉标记**（保持地图纯净）
- 当选中高台角色（如狙击手）时，3 个高台格显示**蓝色半透明高亮**（`Color(0.4, 0.5, 0.7, 0.35)`）
- 实现方式：`deploy_slot.gd` 中的 `_check_high_slot_highlight()` 每帧检查当前选中角色类型

## 四、部署逻辑（deploy_slot.gd + deploy_panel.gd）

### 放置规则（当前版本）

1. **地面角色**（ground deploy_type）：可部署在任意 **ground** 类型的 DeploySlot 上（即路径格子上）
2. **高台角色**（high deploy_type，如狙击手）：只能部署在 **high** 类型的 DeploySlot 上
3. **地雷**（mine）：只能部署在 **ground** 类型的 DeploySlot 上
4. **唯一限制**：必须在路径格（ground）或高台格（high）上，不能放在空白区域

### 关键代码改动

**`deploy_slot.gd` — `try_deploy()`**
```gdscript
# 类型检查已取消（原来会阻止地面角色放在"错误"的格子上）
# 现在只检查是否已被占用
func try_deploy(character_scene: PackedScene, cost: int, type: String) -> bool:
    if is_actually_occupied():
        return false
    # 不再检查 type != slot_type
```

**`deploy_panel.gd` — `can_deploy_on_slot()`**
```gdscript
func can_deploy_on_slot(slot_type: String) -> bool:
    if mine_selected:
        return slot_type == "ground" and Economy.can_afford(MINE_COST)
    if selected_char_id == "":
        return false
    for data in char_buttons:
        if data.id == selected_char_id:
            # 高台角色只能放 high 槽，地面角色只能放 ground 槽
            if data.deploy_type == "high" and slot_type != "high":
                return false
            if data.deploy_type == "ground" and slot_type != "ground":
                return false
            var cost = data.cost
            if TimeCycle.current_phase == TimeCycle.Phase.DUSK:
                cost /= 2
            return Economy.can_afford(cost)
    return false
```

### 选中后行为

- 放置成功后：**自动取消选中**（`selected_char_id = ""`）
- 放置成功后：**隐藏范围预览光圈**（`_hide_range_preview()`）
- 实现位置：`deploy_panel.gd` 的 `on_slot_clicked()` 回调
